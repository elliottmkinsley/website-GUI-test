# Create Headshot WebP Variants

Each headshot has three files on disk:

```
Images/People/<basename>.jpg                     <- the base image (or .png)
Images/People/variants/card/<basename>.webp      <- smaller WebP for homepage cards
Images/People/variants/team/<basename>.webp      <- larger WebP for the Our Team page
```

The site's `<picture>` element prefers the WebP variants because they're smaller and load faster, but it falls back to the original `imageSrc` if a variant is missing. So skipping variants doesn't break anything — it just means the page is slightly heavier.

These variants are **not** auto-generated by anything in the repo. You produce them once when you add a new person.

## When you need to do this

Whenever you add a new headshot to `Images/People/`. If you reuse `Images/People/blank-headshot.png`, no work is needed — its variants already exist.

## Recommended target sizes

| Variant | Approximate dimensions | Aspect | Quality |
|---|---|---|---|
| `variants/card/<basename>.webp` | ~360 x 420 px | 6:7 portrait | 80% |
| `variants/team/<basename>.webp` | ~600 x 720 px | 5:6 portrait | 82% |

Final file sizes typically end up around 25-50 KB for the card variant and 60-110 KB for the team variant.

## Tool option 1 — Squoosh (no install required)

[Squoosh.app](https://squoosh.app/) is a free in-browser converter from Google.

1. Open Squoosh.
2. Drop the original headshot in.
3. Use the resize panel to set the target dimensions (e.g. 360 x 420 for the card variant).
4. Pick "WebP" as the output format. Set quality around 80.
5. Download. Rename to `<basename>.webp` and save to `Images/People/variants/card/`.
6. Repeat for the team variant at 600 x 720.

## Tool option 2 — Photoshop or GIMP

Resize the image to the target dimensions, then "Export As" or "File > Export" to WebP at quality 80-82.

## Tool option 3 — ffmpeg (one-liner)

If you have ffmpeg installed:

```powershell
ffmpeg -i "Images/People/Jane_D.jpg" -vf "scale=360:420" -c:v libwebp -quality 80 "Images/People/variants/card/Jane_D.webp"
ffmpeg -i "Images/People/Jane_D.jpg" -vf "scale=600:720" -c:v libwebp -quality 82 "Images/People/variants/team/Jane_D.webp"
```

## Naming rules

The variant filename's basename **must match the original exactly**, including case. If your headshot is `Images/People/Jane_D.jpg`, both variants must be `Jane_D.webp` (not `jane_d.webp`).

Why: `JS/team-data.js` builds the variant URL by stripping the extension from the JSON's `imageSrc` and inserting `variants/<card|team>/`. Case mismatch can break the variant on case-sensitive web hosts even if it works locally on Windows.

## Verifying

Open the homepage and the Our Team page in a browser. Open DevTools > Network and filter by "webp." You should see `<basename>.webp` requests succeeding (HTTP 200). If they 404, the runtime falls back to the original — the site still works, but you're missing the size win.

## Skipping variants entirely

If you can't produce WebP files, just leave them out. The base image is used everywhere. Plan to come back and add them later for a performance polish pass.

## Bulk-generating variants for existing photos

There's no in-repo script for batch generation. If you ever need to regenerate every variant (for example after a brand refresh that changed crop standards), the recommended approach is:

1. Stand up a one-off Node or Python script using `sharp` or `Pillow` to walk `Images/People/*.jpg`.
2. For each file, write `variants/card/<basename>.webp` at 360 x 420 and `variants/team/<basename>.webp` at 600 x 720.
3. Run once, commit the result, then delete the script (we don't keep build tooling in the repo).
